# Pipeline Intake Checklist

Use this checklist before drafting a new pipeline or when scoping a new dataset.

The goal is to identify the minimum missing information needed to create one
correct pipeline for one dataset.

## Working Rule

Do not ask the user every question by default.

Instead:

1. compare the request against this checklist
2. infer whatever is safely inferable from repo context
3. identify the smallest set of missing, decision-critical items
4. ask only for those missing items if needed

## Core Intake Questions

### 1. Dataset identity

- What is the dataset supposed to be?
- Is the requested output exactly one dataset?
- What should the `dataset_id` likely be?
- What data class does it belong to?
  - `RAW`
  - `EXT`
  - `STAGE`
  - `MODEL_IN`
  - `MODEL_OUT`
  - `SERVE`
  - `REF`

### 2. Canonical output

- What is the final reusable artifact?
- What is the canonical output path or storage location?
- Is the final dataset a file, folder, table, tile set, or another stable unit?
- What files are part of the dataset versus just temporary intermediates?

### 3. Grain

- What does one row, cell, feature, or record represent?
- Is the grain unambiguous?

### 4. Upstream sources

- What source datasets are required?
- Are they already staged locally?
- Are they external, internal, or generated by prerequisite pipelines?

### 5. Spatial and temporal scope

- What years, dates, or time range are included?
- What region, tiles, counties, states, or CRS are relevant?

### 6. Schema and expected fields

- What fields must the final dataset contain?
- Which fields are copied directly from source?
- Which fields are derived?

### 7. Prerequisite work

- Does this dataset depend on an upstream reusable dataset?
- Should that upstream work be a separate pipeline referenced via `requires_pipelines`?

### 8. Runtime and environment

- Will this run locally, on HPCC, or both?
- Does it require module-managed binaries or special dependencies?
- Is the workload likely to be memory-heavy or IO-heavy?

### 9. Data-dictionary requirements

- Does a corresponding data-dictionary entry already exist?
- If not, what dictionary entry should be created alongside the pipeline?

### 10. Provenance and attribution

- What prior work, papers, or external sources should be cited?
- What provenance must be recorded at:
  - dataset level
  - method level
  - feature level

## Decision Output

After using this checklist, the AI should be able to state:

- what the one target dataset is
- whether enough information exists to draft the pipeline
- what the smallest missing information is, if any
- whether a corresponding data-dictionary entry must be created

## Ask Only If Missing

The most common user-facing questions worth asking are:

- What is the final dataset supposed to contain?
- What should the grain be?
- What upstream dataset or source should be treated as canonical?
- What exact years or spatial scope are included?
- What should be documented as provenance or prior work?
